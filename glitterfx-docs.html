Cinematic particle backgrounds, by configuration.

Drop GlitterFX on any HTML container and turn it into a living atmosphere — sparkling glitter, drifting embers, swirling galaxies, splashing waves. 26 effects, 17 palettes, 6 blur modes. ~110 KB. Built on Three.js.

Install

GlitterFX has one peer dependency: Three.js (r128+). Use a CDN for quick prototypes, or npm for production.

CDN

<script src="https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js"></script>
<script src="https://cdn.jsdelivr.net/gh/your-username/glitterfx@latest/dist/glitterfx.js"></script>

NPM

npm install three glitterfx

Quick start

Three lines and a container. Anything inside the container is automatically promoted above the canvas — you don't need to manage z-index.

<section id="hero" style="height: 100vh">
  <h1>Welcome to the future</h1>
</section>

<script>
  new GlitterFX(document.getElementById('hero'), {
    effect:  'emerald-shimmer',
    palette: 'emerald-gold',
    haze:    0.4,
  });
</script>
Heads up

The library expects THREE on the global window object. If you're using a bundler with named imports, do window.THREE = THREE before importing GlitterFX.

Core concepts

GlitterFX builds atmosphere from four independent dimensions. Mix and match — most combinations look great.

Effect

The motion model: how particles spawn, move, and recycle. emerald-shimmer drifts radially toward the camera; dancing-waves arcs in fountains under gravity; curl-flow rides a divergence-free vector field. 26 to choose from.

Palette

The color set: a list of hex colors with weights. Decoupled from the effect, so any motion can wear any color. Pass a name ('magma') or a custom array (['#ff0000', '#00ff00']).

Blur mode

The focal style: where the canvas is sharp vs blurred. Six modes including center-focus (sharp middle, blurred edges) and vignette-blur (the opposite). Combine with the blur strength slider.

Haze

An optional atmospheric glow layer behind the particles — a soft radial halo in any color. Adds depth without GPU cost (it's a CSS gradient, not a shader pass).

The 17 palettes

Palettes are independent of effects. Apply any palette to any effect via fx.update({ palette: 'magma' }).

The 6 blur modes

Blur is a separate dimension from effect choice. Set blurMode and blur (strength in px) independently.

ModeEffect on the canvasImplementation
noneNo blur. Sharpest possible particles.—
uniformEven blur across the whole canvas.CSS filter: blur()
lensMild blur + slight saturation/brightness boost. Out-of-focus lens feel.CSS filter chain
motionVertical directional softness. Suggests speed/wind.Filter + masked overlay
center-focusSharp center, blurred edges. Cinematic.Backdrop-filter + radial mask
vignette-blurBlurred center, sharp edges. Inverse cinematic.Backdrop-filter + radial mask

API

Constructor

Mount a particle effect inside any HTMLElement.

const fx = new GlitterFX(container, config);

The library handles container styling (position: relative, overflow: hidden) and child layering automatically.

Instance methods

MethodDescription
update(patch)Patch any config field live. Density/palette/effect rebuild; everything else applies instantly.
setEffect(name)Switch to a different effect; adopts that effect's defaults.
setScale(scale)Coupled size + density (zoom in/out without clutter).
loadConfig(url)Fetch a JSON config from a URL and apply it.
start() / stop()Pause and resume the animation loop.
destroy()Remove the canvas, dispose WebGL resources, drop listeners.

Static methods

MethodReturns / does
GlitterFX.listEffects()Array of all registered effect IDs.
GlitterFX.getDefaults(name)An effect's default config object.
GlitterFX.registerEffect(name, def)Add a custom effect at runtime.
GlitterFX.listPalettes()Array of all palette names.
GlitterFX.getPalette(name){ colors, weights } for a palette.
GlitterFX.registerPalette(name, colors, weights)Add a custom palette at runtime.
GlitterFX.listBlurModes()Array of valid blurMode values.

Recipes

Hero section

The most common pattern. Mount on a viewport-tall section.

new GlitterFX(document.querySelector('#hero'), {
  effect:    'emerald-shimmer',
  haze:      0.4,
  blur:      2,
  blurMode:  'lens',
});

Full-page background

Mount on document.body for a whole-viewport atmosphere. All page content sits above the canvas naturally.

new GlitterFX(document.body, {
  effect:    'cosmic-dust',
  haze:      0.6,
  blurMode:  'center-focus',
  blur:      4,
});

Switch effects on scroll

Use IntersectionObserver to swap effects as the user moves through sections.

const fx = new GlitterFX(bg, { effect: 'star-field' });

document.querySelectorAll('.scroll-section').forEach(section => {
  new IntersectionObserver(([entry]) => {
    if (entry.isIntersecting && entry.intersectionRatio > 0.5) {
      fx.setEffect(section.dataset.effect);
    }
  }, { threshold: [0.5] }).observe(section);
});

Pause when offscreen

Battery and CPU friendly. Pause the loop when the host element scrolls out of view.

const fx = new GlitterFX(section, { effect: 'aurora-veil' });

new IntersectionObserver(([entry]) => {
  entry.isIntersecting ? fx.start() : fx.stop();
}).observe(section);

Extending

Add a custom palette

Register your brand colors and use them with any effect.

GlitterFX.registerPalette(
  'sunset-pier',
  ['#ff6b35', '#f7c548', '#ffe5d4', '#3d5a80', '#293241'],
  [0.30, 0.25, 0.20, 0.15, 0.10]   // optional weights
);

new GlitterFX(el, {
  effect:  'curl-flow',
  palette: 'sunset-pier',
});

Add a custom effect

Effects are specs passed to buildKernelEffect. Define a spawn() for where particles start and a step() for how they move per frame.

GlitterFX.registerEffect('drifting-orbs', {
  defaults: {
    background:  'radial-gradient(circle at 50% 50%, #0a0a18 0%, #000 100%)',
    density:     0.7,
    size:        1.0,
    brightness:  0.9,
    speed:       0.4,
    blur:        2,
    blurMode:    'lens',
    haze:        0.3,
    hazeColor:   '#7a8aff',
    palette:     'cosmic',
  },

  build(ctx) {
    return buildKernelEffect(ctx, {
      countBase: 800,
      cameraPos: [0, 0, 25],
      useLifetime: false,

      spawn(k) {
        const i = k.i, p = k.positions;
        p[i*3]   = (Math.random() - 0.5) * 40;
        p[i*3+1] = (Math.random() - 0.5) * 25;
        p[i*3+2] = (Math.random() - 0.5) * 6;
      },

      step(k) {
        const i = k.i, p = k.positions;
        p[i*3]   += Math.sin(k.time + i) * k.dt;
        p[i*3+1] += Math.cos(k.time + i*0.7) * k.dt;
      },
    });
  },
});

new GlitterFX(el, { effect: 'drifting-orbs' });

For the full kernel spec — useLifetime, twinkle, spikes, fog, custom shaders — see CONTRIBUTING.md.
